# Інструкція з генерації документації для проєкту

Ця інструкція пояснює, як згенерувати документацію для проєкту за допомогою інструмента Sphinx.

## 1. Встановлення залежностей

Для створення документації необхідно встановити Sphinx та інші потрібні пакети. Для цього:

1.1. Встановіть Sphinx через pip:

```bash
pip install sphinx
```
## 2. Ініціалізація документації

2.1. Відкрийте термінал у кореневій директорії проєкту і запустіть команду:
```
sphinx-quickstart docs
```
2.2. Відповідайте на запитання, що з'являються в процесі налаштування (вибір мови, назв проекту тощо). Це створить основну структуру папок і файлів для документації в директорії docs.

## 3. Заповнення основного файлу документації

3.1. Відкрийте файл docs/source/index.rst і заповніть його основною інформацією про ваш проєкт, як описано в попередньому розділі.
```
.. SeriesBot documentation master file, created by
   sphinx-quickstart on Fri Apr 18 16:11:17 2025.
   You can adapt this file completely to your liking, but it should at least
   contain the root `toctree` directive.

Welcome to SeriesBot's documentation!
======================================

.. toctree::
   :maxdepth: 2
   :caption: Зміст:

   modules
```

## 4. Генерація документації

4.1. Щоб згенерувати документацію в HTML форматі, використовуйте команду:

```
make html
```
Ця команда створить HTML-файли документації в директорії docs/build/html.

## 5. Перегляд документації

5.1. Відкрийте файл docs/build/html/index.html у браузері, щоб переглянути згенеровану документацію.

## 6. Додавання документації до репозиторію
6.1. Після генерації документації, згенерований архів з документацією можна додати до репозиторію.

6.2. Створіть архів з документацією, використовуючи команду:
```
zip -r SeriesBotDocs.zip docs/build/html
```
6.3. Додайте архів до репозиторію:

```
git add SeriesBotDocs.zip
git commit -m "Added generated documentation archive"
git push
```
## 7. Інші корисні команди
- Оновлення документації: Якщо ви вносите зміни в код або файли документації, просто перезапустіть команду make html, щоб перегенерувати документацію.

- Генерація документації для інших форматів: Ви можете також генерувати документацію для інших форматів, наприклад, PDF чи LaTeX, використовуючи відповідні команди make.
